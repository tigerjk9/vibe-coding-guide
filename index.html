고급 제작 가이드

"작동하는 초안"에서 "수업에 쓰는 완성품"으로 끌어올리는 방법

📊 입문 바이브코딩 vs 고급 바이브코딩

구분 입문 (지금까지) 고급 (이번 연수)
데이터코드에 문제를 직접 입력문제를 배열/JSON으로 분리 → 추가·교체 쉬움
화면한 화면 고정시작 → 문제 → 결과 화면 전환
기록새로고침하면 초기화localStorage로 점수·진행 저장
디자인PC에서만 보기 좋음태블릿·전자칠판까지 반응형
접근성고려 안 함큰 글자·넓은 버튼·키보드·읽어주기
배포Canvas 미리보기로 끝고정 URL로 배포 → 재사용·공유

🧭 제작 워크플로 7단계

⚡ 완성도를 높이는 5가지 기법

① 데이터 분리

문제·정답을 코드 맨 위 배열로 빼두면, 코딩을 몰라도 내용만 바꿔 재사용할 수 있어요.

② 상태 저장

localStorage로 최고 점수·진행 상황을 저장. 새로고침해도 이어집니다.

③ 반응형

"전자칠판(큰 화면)과 학생 태블릿에서 모두 잘 보이게"라고 요청하세요.

④ 접근성

저학년엔 큰 글자·넓은 버튼·읽어주기(TTS). "1~2학년도 혼자 쓸 수 있게".

⑤ 안전 설계

서버 없이 브라우저 안에서만 동작 → 개인정보 수집·전송 없음. 가장 안전.

＋ 한 파일 유지

HTML·CSS·JS를 한 파일로. 복사·배포·공유가 가장 쉬워집니다.
